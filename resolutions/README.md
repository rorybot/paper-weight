# Resolutions

Small, single-issue files recording how we actually fixed (or diagnosed) a real
recurring problem on this project — device networking, kiosk/browser state,
build/deploy weirdness, etc. Not a spec, not a kanban card: just "here's what
was actually wrong and what we did about it," so a future session greps here
before re-deriving the same diagnosis from scratch.

One file per distinct problem. Name it after the symptom, not the fix.
Keep each one short: symptom, root cause, fix/command, and status
(resolved / mitigated / open).
